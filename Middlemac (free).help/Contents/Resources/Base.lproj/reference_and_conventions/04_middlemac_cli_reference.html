Middlemac command line reference

Learn to master the Middlemac command line interface.

Getting help and identifying valid targets

If you run Middlemac with an invalid target, with no parameters, or with the -h or --help switch, Middlemac will display its documentation.

Middlemac command signature

./middlemac.rb TARGET... | all [--verbose | -v] [--quiet | -q] [--help | -h] [--server | -s]

TARGET...
One or more valid targets as configured in config.rb. If you specify a mix of valid and invalid targets, Middlemac will fail with an explanation.
all
The all target will build all of the targets that are configured in config.rb. If specified with valid targets, the valid targets will be ignored in the command, but built as they are included in all. However all specified with invalid targets will fail with an explanation.
--verbose
-v
Executes Middleman in verbose mode for each build target.
--quiet
-q
Silences Middleman output, even if –verbose is specified.
--help
-h
Displays help and doesn’t process files or run server.
--server
-s
Runs Middleman in server mode. Server target is undefined if more than one or all is specified!
